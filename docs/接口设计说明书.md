## 1. 通用约定

### 1.1 Base URL

```
https://{host}/api/v1
```

### 1.2 鉴权

除 `GET /health`、`POST /auth/register`、`POST /auth/login` 外，所有请求须携带：

```
Authorization: Bearer <token>
```

Token 由注册或登录返回；服务端查 `user_sessions` 解析 `user_id`。未授权返回 `401`。

#### `POST /auth/register`

无需鉴权。

**请求**：

```json
{
  "username": "alice",
  "password": "secret123",
  "display_name": "小明",
  "birthday": "2000-01-01"
}
```

`display_name` 默认同 `username`；`birthday` 可选。

**响应** `201`：

```json
{
  "data": {
    "token": "doki_xxxxxxxx",
    "user": {
      "id": "uuid",
      "username": "alice",
      "display_name": "小明",
      "birthday": "2000-01-01",
      "max_proactive_per_day": 20
    }
  }
}
```

#### `POST /auth/login`

**请求**：`{ "username", "password" }`

**响应** `200`：格式同注册。

### 1.3 通用响应格式

**所有 REST 端点**（含 `GET /health`）均使用统一 envelope。

**成功（有 body）**：

```json
{
  "data": { }
}
```

`data` 可为 object、array 或 string。

**成功（无业务 body）**：

用于 DELETE、部分 PATCH 等无返回内容的操作。固定返回 `200`：

```json
{
  "data": "ok"
}
```

**错误**：

```json
{
  "error": {
    "code": "INVALID_TOKEN",
    "message": "人类可读说明"
  }
}
```

### 1.4 错误码

| HTTP | code | 说明 |
|------|------|------|
| 400 | `BAD_REQUEST` | 参数无效 |
| 401 | `INVALID_TOKEN` | Token 无效或已注销 |
| 401 | `INVALID_CREDENTIALS` | 用户名或密码错误 |
| 409 | `USERNAME_TAKEN` | 用户名已存在 |
| 404 | `NOT_FOUND` | 资源不存在 |
| 413 | `PAYLOAD_TOO_LARGE` | 上传超限 |
| 415 | `UNSUPPORTED_MEDIA` | 图片类型不支持 |
| 500 | `INTERNAL_ERROR` | 服务端错误 |
| 503 | `LLM_UNAVAILABLE` | LLM 调用失败 |

## 2. REST API

### 2.1 健康检查

#### `GET /health`

无需鉴权。

**响应** `200`：

```json
{
  "data": "ok"
}
```

---

### 2.2 认证

见 §1.2 `POST /auth/register`、`POST /auth/login`。

### 2.3 用户档案

#### `GET /me`

**响应** `200`：

```json
{
  "data": {
    "id": "uuid",
    "username": "alice",
    "display_name": "小明",
    "birthday": "2000-01-01",
    "max_proactive_per_day": 20
  }
}
```

#### `PATCH /me`

**请求**：

```json
{
  "display_name": "小明",
  "birthday": "2000-01-01",
  "max_proactive_per_day": 15
}
```

字段均可选。

---

### 2.4 角色与会话

#### `GET /characters`

角色列表（MVP 全量返回，无分页）。

**响应** `200`：

```json
{
  "data": [
    {
      "id": "uuid",
      "name": "角色名",
      "avatar_url": "/api/v1/characters/{id}/avatar"
    }
  ]
}
```

#### `GET /conversations`

会话列表。

**响应** `200`：

```json
{
  "data": [
    {
      "id": "uuid",
      "character_id": "uuid",
      "character_name": "角色名",
      "status": "active",
      "last_message": {
        "content": "最近一条",
        "created_at": "2026-07-08T08:00:00Z",
        "role": "character"
      },
      "current_activity": "上课"
    }
  ]
}
```

`current_activity` Phase 2 展示；MVP 可返回 null。

#### `POST /conversations`

创建或获取与某角色的会话（幂等）。

**请求**：

```json
{ "character_id": "uuid" }
```

**响应** `200` / `201`：

```json
{
  "data": {
    "id": "uuid",
    "character_id": "uuid",
    "status": "active",
    "first_contact_done": false
  }
}
```

首次创建时触发破冰（见详细设计 §12）。

---

### 2.5 消息

#### `GET /conversations/{id}/messages`

分页历史消息，按时间升序。

**Query**：`?before={message_id}&limit=50`（默认 50，最大 100）

**响应** `200`：

```json
{
  "data": {
    "messages": [
      {
        "id": "uuid",
        "role": "user",
        "content": "今天好累",
        "content_type": "text",
        "turn_id": "uuid",
        "seq_in_turn": 0,
        "image_url": null,
        "reply_to_id": null,
        "read_at": null,
        "created_at": "2026-07-08T08:00:00Z"
      }
    ],
    "has_more": false
  }
}
```

`content_type`：`text` | `image`

#### `POST /conversations/{id}/messages`

发送文本消息。实际回复经 WebSocket 推送。

**请求**：

```json
{
  "content": "今天好累",
  "client_msg_id": "optional-uuid-for-dedup"
}
```

**响应** `202`：

```json
{
  "data": {
    "id": "uuid",
    "turn_id": "uuid",
    "created_at": "2026-07-08T08:00:00Z"
  }
}
```

#### `POST /conversations/{id}/messages/image`

`multipart/form-data`：

| 字段 | 类型 | 必填 |
|------|------|------|
| `image` | file | 是 |
| `caption` | string | 否 |

限制：JPEG/PNG/WebP，最大 10MB。

**响应** `202`：同文本消息。

#### `GET /messages/{id}/image`

获取图片（需鉴权，校验会话归属）。

**响应**：图片二进制，`Content-Type: image/*`

## 3. WebSocket

### 3.1 连接

```
wss://{host}/api/v1/ws
```

连接时携带 Header `Authorization: Bearer <token>`（与 REST 相同）。

连接成功后服务端发送：

```json
{
  "type": "connected",
  "payload": { "user_id": "uuid" }
}
```

### 3.2 客户端 → 服务端

#### 订阅会话

```json
{
  "type": "subscribe",
  "payload": { "conversation_id": "uuid" }
}
```

#### 发送消息（可选，与 REST 二选一）

```json
{
  "type": "send_message",
  "payload": {
    "conversation_id": "uuid",
    "content": "你好",
    "client_msg_id": "optional"
  }
}
```

#### 心跳

```json
{ "type": "ping" }
```

服务端回复 `{ "type": "pong" }`。建议 30s 间隔。

### 3.3 服务端 → 客户端

#### 新消息气泡

```json
{
  "type": "message",
  "payload": {
    "id": "uuid",
    "conversation_id": "uuid",
    "role": "character",
    "content": "怎么了？",
    "content_type": "text",
    "turn_id": "uuid",
    "seq_in_turn": 0,
    "created_at": "2026-07-08T08:00:01Z"
  }
}
```

#### 正在输入

客户端收到后更新 **ChatPage AppBar 副标题**为「对方正在输入…」；`active=false` 时恢复（Phase 2 可恢复为活动状态文案）。不在消息气泡区展示。

```json
{
  "type": "character_typing",
  "payload": {
    "conversation_id": "uuid",
    "active": true
  }
}
```

#### 已读回执

```json
{
  "type": "message_read",
  "payload": {
    "conversation_id": "uuid",
    "message_ids": ["uuid1", "uuid2"],
    "read_at": "2026-07-08T08:00:00.5Z"
  }
}
```

#### 回合取消（投递被打断）

```json
{
  "type": "turn_cancelled",
  "payload": {
    "conversation_id": "uuid",
    "turn_id": "uuid"
  }
}
```

#### 会话状态变更

```json
{
  "type": "conversation_status",
  "payload": {
    "conversation_id": "uuid",
    "status": "paused"
  }
}
```

#### 错误

```json
{
  "type": "error",
  "payload": {
    "code": "BAD_REQUEST",
    "message": "说明"
  }
}
```

## 4. 推送（FCM）

### 4.1 离线主动消息

App 后台时，主动消息通过 FCM 送达。

**Notification payload（MVP 通用文案）**：

```json
{
  "notification": {
    "title": "Dokidoki",
    "body": "你有一条新消息"
  },
  "data": {
    "conversation_id": "uuid",
    "character_id": "uuid",
    "type": "proactive_message"
  }
}
```

Phase 2 / P4：通知文案人格化。

### 4.2 设备注册

#### `POST /devices`

**请求**：

```json
{
  "fcm_token": "string",
  "platform": "android"
}
```

## 5. 时序示例：Burst Chat

```
Client                          Server
  | POST /messages "今天"           |
  |------------------------------->|
  | POST /messages "好累"           |
  |------------------------------->|
  |        (静默 2s)                |
  |<-------------------------------| message_read (忙碌时)
  |<-------------------------------| character_typing active=true (AppBar)
  |<-------------------------------| message "怎么了？"
  |<-------------------------------| message "发生什么事了"
  |<-------------------------------| character_typing active=false (AppBar 恢复)
```
